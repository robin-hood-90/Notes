## ALB vs NLB vs GWLB

| Feature | ALB (Layer 7) | NLB (Layer 4) | GWLB (Layer 3) |
|:--------|:-------------:|:-------------:|:---------------:|
| **Protocol** | HTTP, HTTPS, gRPC, WebSocket | TCP, UDP, TLS | IP (Geneve encapsulation) |
| **Content-based routing** | Yes (host header, path pattern, query string, HTTP method) | No (only port-based) | No |
| **Static IP** | No (unless Global Accelerator) | Yes (per AZ EIP) | Yes (per AZ EIP) |
| **TLS termination** | Yes (ACM cert) | Yes (ACM cert) | No (pass-through) |
| **WAF** | Yes (Web ACL) | No | No |
| **Authentication** | Cognito, OIDC, LDAP | No | No |
| **Health checks** | HTTP/HTTPS | TCP, HTTP, HTTPS | Health check per target |
| **Speed** | Lower | Highest (millions of requests/sec) | Throughput-focused |
| **Use case** | Microservices, containers, WebSocket | Game servers, DNS, TCP/UDP | Firewall appliances |

### ALB-specific features

```text
Cross-zone load balancing: distributes traffic across all targets in all AZs.
  - Enabled by default for ALB.
  - For NLB: disabled by default (enable manually).

Request tracing: X-Amzn-Trace-Id header for end-to-end request tracking.

Sticky sessions (session affinity):
  - Duration-based cookie (AWSALB): generated by ALB, TTL 1s–7d.
  - Application-based cookie: custom cookie name, generated by the app.

Authentication:
  - Cognito User Pools: authenticate users via Cognito OAuth flow.
  - OIDC: any OpenID Connect-compatible IdP (Okta, Auth0).
  - LDAP: Active Directory via NTLM or Kerberos.

Fixed response and redirect:
  - Fixed response: return static HTTP response (e.g., maintenance page).
  - Redirect: HTTP→HTTPS, domain.com→www.domain.com.
```

## Target Groups and Health Checks

> [!info] Target group
> A target group routes requests to registered targets (EC2 instances, IP addresses, Lambda functions, or ECS tasks) in a specified protocol/port. Each target group has one or more health checks.

```text
Target types:
  - Instance: EC2 instance ID (ALB reaches instance by private IP).
  - IP: ENI private IP (used by ECS with awsvpc mode, NLB for on-prem).
  - Lambda: Lambda function ARN (ALB invokes Lambda and returns the response).

Health check configuration:
  - Protocol: HTTP, HTTPS, TCP.
  - Path: /healthz (for HTTP).
  - Port: traffic port or override port.
  - Interval: 5–300 seconds (default 30).
  - Timeout: 2–120 seconds (default 5).
  - Healthy threshold: 2–10 (default 5).
  - Unhealthy threshold: 2–10 (default 2).
  - Status codes: specify which HTTP codes are healthy (200–399).
  - GRPC: health check path must be /grpc.health.v1.Health/Check.
```

## Listeners and Routing

> [!info] Listener
> A listener checks for connection requests using the configured protocol/port. A listener has default rules (forwared to default target group) and optional rules (forward based on conditions).

```text
Rule conditions:
  - Host header: `api.example.com`, `*.example.com`.
  - Path pattern: `/api/*`, `/images/*.jpg`.
  - HTTP method: GET, POST, PUT, DELETE.
  - Query string: `?version=2` (name/value pair).
  - Source IP: CIDR range.
  - HTTP header: custom header name/value.

Action:
  - Forward: send to a target group (single or weighted).
  - Redirect: HTTP→HTTPS, external URL.
  - Fixed-response: return a static code + body.
  - Authenticate: Cognito or OIDC before forwarding.

Weighted target groups (forward to multiple TGs with weights):
  - Use: canary deployments, A/B testing, blue/green.
  - Example: default TG weight 90, canary TG weight 10.
```

## TLS Termination

> [!info] TLS termination
> ALB and NLB can terminate TLS (HTTPS), decrypt requests, and forward plaintext HTTP to backends. ACM (AWS Certificate Manager) manages certificates. ALB and NLB can also do mutual TLS (mTLS).

```text
TLS version: minimum 1.0, 1.2, or 1.3 (configured via security policy).
Security policy reference:
  - ELBSecurityPolicy-TLS13-1-2-2021-06 (TLS 1.3 + 1.2, strongest).
  - ELBSecurityPolicy-TLS-1-1-2017-01 (TLS 1.1 + 1.2).
  - Avoid: TLS 1.0 (deprecated).

Mutual TLS (mTLS):
  - ALB validates client certificates against a trust store.
  - Trust store: ACM Private CA or custom CA certificate bundle (stored in S3).
  - Verified client certificate: passed as header (X-Forwarded-Client-Cert) to the backend.
```
